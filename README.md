여러 인스턴스(서버)가 있을 때, 각 인스턴스는 자체 메모리에 소켓 연결을 관리한다.
소켓은 특정 서버 프로세스에 물리적으로 연결되어 있기 때문에, 다른 인스턴스에서는 그 소켓에 대한 접근 자체가 불가능하다. 결국, 여러 인스턴스에서 띄운 소켓들간의 메모리 공유가 이루어지지 않아 오류를 초래한다.

이 프로젝트는 이런 문제에 대한 여러 해결 방법을 고안 후 테스트해보는 목적이다.

## 방법 1: 같은 IP는 항상 같은 앱으로

<!-- nginx -->
upstream backend {
  ip_hash;  # 같은 IP는 항상 같은 앱으로
  server app-1:3000;
  server app-2:3000;
}

먼저 가장 간단한 해결 방법으로는 같은 IP의 유저는 항상 같은 앱으로 보내는 것이다. 하지만 이러면 여러 같은 ip를 가진 사람들이나 같은 room에 참여한 사람들이 같은 app으로 몰리는 문제(로드밸런싱 불균등), WiFi ↔ LTE 전환 시 IP 변경 등 상황에게서 문제들이 생길 것이다.

## 방법 2: Redis Adapter로 인스턴스 메모리 분리 문제 해결

따라서 현재 구현은 Socket.IO Redis adapter를 사용한다.
기존 socket emit을 redis adapter로 승격하여서, 어댑터가 가로채서 Redis에 publish하게 만들면, 모든 인스턴스가 수신 → 각자 broadcast가 가능해진다.

- 각 앱 인스턴스는 Redis pub/sub으로 이벤트를 동기화한다.
- 따라서 room join/message가 서로 다른 앱 인스턴스 간에도 공유된다.

기존 Socket.io 코드 수정 없이 스케일아웃이 가능하다는 점도 redis adapter의 매력적인 부분중 하나이다.

일단 단일 인스턴스에서의 여러 app들간의 socket 정보 공유가 정상 동작하는걸 확인 할 수 있다.
